10.5. Tipos de joints en URDF – Revolute, Continuous y Prismatic

Aprenderás sobre los diferentes tipos de joints disponibles en URDF: revolute, continuous, prismatic y sus características.

📘 Notas de la clase

🔧 1. Referencia a la documentación

  • Aunque la documentación de URDF está en el sitio de ROS 1, es válida también para ROS 2.

  • Buscar en Google: urdf xml site:wiki.ros.org

  • URL recomendada:
    http://wiki.ros.org/urdf/XML

Desde allí puedes consultar:

  • Elementos válidos dentro de <link>: visual, geometry, material, etc.

  • Tipos de joints y sus configuraciones.

🔗 2. Tipos de joints

🔹 Fixed

  • Sin movimiento relativo entre los links.

  • Sintaxis simple:

    <joint name="..." type="fixed">

🔹 Revolute

  • Movimiento de rotación limitada sobre un eje fijo.

  • Requiere:

    • axis: eje de rotación (xyz)

    • limit: rango de rotación en radianes

Ejemplo (rotación sobre eje Z):

<joint name="base_second_joint" type="revolute">
<parent link="base_link"/>
<child link="second_link"/>
<origin xyz="0 0 0.2" rpy="0 0 0"/>
<axis xyz="0 0 1"/>
<limit lower="-1.57" upper="1.57" effort="100" velocity="100"/>
</joint>

📐 1.57 rad = 90°
Valores en radianes: π ≈ 3.14

🔹 Continuous

  • Como revolute, pero sin límites.

  • Ideal para ruedas u objetos que giran libremente.

Ejemplo:

<joint name="wheel_joint" type="continuous">
<parent link="chassis"/>
<child link="wheel"/>
<origin xyz="..." rpy="..."/>
<axis xyz="0 0 1"/>
</joint>

🔁 No se necesita el tag limit.

🔹 Prismatic

  • Movimiento lineal (deslizamiento) entre dos links.

  • Requiere:

    • axis: dirección del desplazamiento

    • limit: distancia máxima/mínima permitida

Ejemplo (deslizamiento en eje X):

<joint name="slider_joint" type="prismatic">
<parent link="base_link"/>
<child link="slider_link"/>
<origin xyz="0 0 0" rpy="0 0 0"/>
<axis xyz="1 0 0"/>
<limit lower="0.0" upper="0.2" effort="100" velocity="100"/>
</joint>

🧭 3. Orden recomendado para construir joints

  1. Crear los links.

  2. Definir el joint:

    • parent y child

    • origin → posición del segundo link respecto al primero

  3. Ajustar el axis si aplica (rotación o deslizamiento).

  4. Incluir limit si es revolute o prismatic.

  5. Finalmente, ajustar el origin del visual del link hijo para que coincida visualmente.

⚠️ Evita tocar el visual antes de tener el TF correcto.

💾 Código trabajado / recursos

Tipos de joints tratados:

  • type="fixed"

  • type="revolute" + axis + limit

  • type="continuous" + axis

  • type="prismatic" + axis + limit
