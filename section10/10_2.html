10.2. Añadir color a un link URDF con etiquetas material

Aprenderás a añadir colores y propiedades visuales a los links de tu URDF usando las etiquetas material.

📘 Notas de la clase

🎨 1. Color por defecto en RViz

  • Cuando no se especifica color para un link, RViz lo mostrará en rojo por defecto.

  • Esto puede dificultar la visualización cuando se agregan múltiples links.

🧾 2. Crear una etiqueta material

  • La etiqueta material se coloca dentro del bloque robot, pero fuera de cualquier link.

  • Cada material debe tener un atributo name y contener una sub-etiqueta color con los valores rgba.

Ejemplo de material verde:

<material name="green">
<color rgba="0 0.5 0 1"/>
</material>

📌 RGBA: Red, Green, Blue, Alpha (transparencia). Los valores van de 0 a 1.

🧩 3. Aplicar el material a un link

  • La referencia al color se hace dentro del bloque visual, después de la etiqueta origin.

Ejemplo:

<visual>
<geometry>
<box size="0.6 0.4 0.2"/>
</geometry>
<origin xyz="0 0 0.1" rpy="0 0 0"/>
<material name="green"/>
</visual>

⚠️ Importante: La etiqueta material no debe colocarse dentro de geometry ni fuera de visual.

🌈 4. Cambiar colores y reutilizar materiales

  • Puedes crear múltiples etiquetas material para diferentes colores.

  • Cambia el valor de rgba y su nombre para representar el color deseado.

Ejemplo de material azul:

<material name="blue">
<color rgba="0 0 0.5 1"/>
</material>

  • Para usarlo, cambia también la referencia en el visual:

    <material name="blue"/>

💾 Código trabajado / recursos

Ejemplo completo:

< ?xml version="1.0" ? >
<robot name="my_robot">
<material name="blue">
<color rgba="0 0 0.5 1"/>
</material>
<link name="base_link">
<visual>
<geometry>
<box size="0.6 0.4 0.2"/>
</geometry>
<origin xyz="0 0 0.1" rpy="0 0 0"/>
<material name="blue"/>
</visual>
</link>
</robot>
