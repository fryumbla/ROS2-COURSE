10.3. Añadir un segundo link y primer joint en URDF

Aprenderás a crear múltiples links y conectarlos con joints para construir una cadena cinemática en tu URDF.

📘 Notas de la clase

🔗 1. Añadir un segundo link

  • Añadir un nuevo <link> con su respectivo <visual>, <geometry> y <origin>.

  • En lugar de una caja, esta vez se usará un cilindro:

    <geometry>
    <cylinder radius="0.1" length="0.2"/>
    </geometry>

  • Al igual que antes, se incluye el tag <origin>:

    <origin xyz="0 0 0" rpy="0 0 0"/>

  • También se define un nuevo material:

    <material name="gray">
    <color rgba="0.5 0.5 0.5 1"/>
    </material>

  • Y se referencia en el visual:

    <material name="gray"/>

⚙️ 2. Importancia de los joints y el error común

  • Si tienes dos links sin joint, ROS no puede determinar la jerarquía y lanza un error:

    Error: failed to find root link. Two root links found: base_link and second_link

🧩 3. Crear un joint entre los links

  • El joint define la relación entre los links, tanto para el modelo como para los TF.

  • Se coloca dentro del robot, como un nuevo tag:

    <joint name="base_second_joint" type="fixed">
    <parent link="base_link"/>
    <child link="second_link"/>
    <origin xyz="0 0 0.2" rpy="0 0 0"/>
    </joint>

🔧 Tipo fixed: los dos links están fijos entre sí (sin movimiento relativo).

🧭 4. Proceso correcto para ubicar los componentes

  1. Poner todos los origin en 0 0 0 al comenzar.

  2. Ajustar primero el origin del joint para alinear correctamente los TF.

    • En este caso, se desea colocar el segundo link encima del primero (la caja base).

    • Como la caja tiene altura 0.2 m → offset de 0.2 en eje Z en el joint:

      <origin xyz="0 0 0.2" rpy="0 0 0"/>

  3. Una vez que el TF es correcto, ajustar el origin del visual para centrarlo sobre su marco.

    • El cilindro tiene una altura (length) de 0.2 → offset de 0.1 en Z:

      <origin xyz="0 0 0.1" rpy="0 0 0"/>

🧠 Recomendaciones clave

  • No modifiques ambos origin al mismo tiempo.

  • Primero alinea bien los TF (joint), luego ajusta el visual.

  • Esto evitará errores comunes de solapamiento o ubicación incorrecta del modelo en RViz.

💾 Código trabajado / recursos

Resumen estructural de los componentes nuevos:

  • Segundo link:

    <link name="second_link">
    <visual>
    <geometry>
    <cylinder radius="0.1" length="0.2"/>
    </geometry>
    <origin xyz="0 0 0.1" rpy="0 0 0"/>
    <material name="gray"/>
    </visual>
    </link>

  • Joint que conecta base_link con second_link:

    <joint name="base_second_joint" type="fixed">
    <parent link="base_link"/>
    <child link="second_link"/>
    <origin xyz="0 0 0.2" rpy="0 0 0"/>
    </joint>
