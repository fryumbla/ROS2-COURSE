10.4. Añadir un tercer link con su joint y visualización precisa

Aprenderás a expandir tu modelo URDF con un tercer link y precisar los parámetros geométricos para una visualización exacta.

📘 Notas de la clase

🧱 1. Crear un tercer link

  • Añadimos un nuevo <link> después del second_link, usando nuevamente un <box>:

    <link name="third_link">
    <visual>
    <geometry>
    <box size="0.1 0.1 0.1"/>
    </geometry>
    <origin xyz="0 0 0" rpy="0 0 0"/>
    <material name="blue"/>
    </visual>
    </link>

⚙️ 2. Conectar el tercer link con un joint

  • Se crea un nuevo joint entre second_link y third_link:

    <joint name="second_third_joint" type="fixed">
    <parent link="second_link"/>
    <child link="third_link"/>
    <origin xyz="0 0 0.2" rpy="0 0 0"/>
    </joint>

📌 0.2 es el largo del cilindro, por lo tanto, el desplazamiento en Z necesario para colocar la caja sobre él.

🎯 3. Ajustar el visual del tercer link

  • El <box> está centrado en su origen por defecto.

  • Para colocarlo justo encima del cilindro, se necesita un offset en z igual a la mitad de su altura (0.05):

    <origin xyz="0 0 0.05" rpy="0 0 0"/>

✅ 4. Orden correcto para posicionar componentes

  1. Crear el nuevo link y su visual, con origin en 0 0 0.

  2. Crear el joint, especificar el parent, child, y ajustar su origin para que el TF esté correcto.

  3. Una vez verificado el TF en RViz, ajustar el origin del visual para alinear visualmente el objeto.

⚠️ No intentes ajustar ambos origins (visual y joint) al mismo tiempo.

💾 Código trabajado / recursos

Resumen de los nuevos componentes:

  • Tercer link (caja):

    <link name="third_link">
    <visual>

    <box size="0.1 0.1 0.1"/>
    </geometry>
    <origin xyz="0 0 0.05" rpy="0 0 0"/>
    <material name="blue"/>
    </visual>
    </link>

  • Joint entre segundo y tercer link:

    <joint name="second_third_joint" type="fixed">
    <parent link="second_link"/>
    <child link="third_link"/>
    <origin xyz="0 0 0.2" rpy="0 0 0"/>
    </joint>
