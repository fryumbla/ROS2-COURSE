6.1. Creación de un servidor de servicio en ROS 2 con Python

Aprenderás a crear un servidor de servicio en ROS 2 utilizando Python, permitiendo que otros nodos soliciten acciones a través de un patrón cliente-servidor.

Documentacion
https://docs.ros.org/en/jazzy/Tutorials/Beginner-CLI-Tools/Understanding-ROS2-Services/Understanding-ROS2-Services.html


📘 Notas de la clase

🔹 Conceptos clave

  • Un servicio en ROS 2 está compuesto por un nombre y una interfaz.

  • Las interfaces de servicios tienen dos partes: request (solicitud del cliente) y response (respuesta del servidor).

  • En este ejemplo se usa la interfaz example_interfaces/srv/AddTwoInts.

🔹 Exploración de la interfaz

  • Comando para ver la interfaz:
    ros2 interface show example_interfaces/srv/AddTwoInts

  • La interfaz contiene:

    • Request: dos enteros a y b

    • Response: un entero sum

  • Los tres guiones (---) separan la solicitud de la respuesta.

🔹 Estructura del código del servidor

  1. Crear un nuevo archivo: add_two_ints_server.py

  2. Hacerlo ejecutable.

  3. En el código:

    • Crear una clase nodo que herede de Node

    • Importar la interfaz de servicio: from example_interfaces.srv import AddTwoInts

    • Crear el servidor de servicio en el constructor con: self.create_service(AddTwoInts, 'add_two_ints', self.callback_add_two_ints)

  4. Definir el callback:

    • Recibe request y response

    • Realiza la suma:
      response.sum = request.a + request.b

    • Registrar en el log lo que se está haciendo:
      Por ejemplo: "3 + 7 = 10"

    • Retornar el objeto response (¡no olvidarlo!).

🔹 Buenas prácticas

  • Nombrar los servicios con verbos (add_two_ints, compute_area, enable_motor, etc.) para reflejar acciones.

  • Dar nombres explícitos a los callbacks: callback_add_two_ints.

  • Validar siempre que el package.xml tenga la dependencia de example_interfaces.

🔹 Prueba desde la terminal (sin código cliente aún)

  • Comando:
    ros2 service call /add_two_ints example_interfaces/srv/AddTwoInts "{a: 3, b: 7}"

  • Resultado esperado:

    • En el servidor: log 3 + 7 = 10

    • En la terminal del cliente: sum: 10

🔹 Consideraciones

  • El servidor debe estar corriendo y en espera para recibir solicitudes.

  • Para peticiones complejas se recomienda implementar un cliente (se verá en la siguiente clase).

  • Olvidar retornar el objeto response en el callback es un error común que causará fallos silenciosos.
