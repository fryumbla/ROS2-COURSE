6.2. Creación de un cliente de servicio en ROS 2 con Python con OOP

Aprenderás a crear un cliente de servicio en ROS 2 utilizando Python con programación orientada a objetos (OOP), permitiendo que tu nodo solicite servicios de forma estructurada y escalable.

📘 Notas de la clase

🔹 Introducción al cliente de servicio en OOP

  • El cliente de servicio se integra dentro de un nodo ROS 2 que sigue el enfoque OOP.

  • Se evita el uso de rclpy.spin_until_future_complete() ya que haría "spin dentro del spin", lo cual genera comportamientos indeseados.

  • Se usa add_done_callback() para manejar la respuesta cuando el servidor responde.

🔹 Estructura del archivo

  1. Crear el archivo: add_two_ints_client.py

  2. Hacerlo ejecutable.

  3. Crear una clase que herede de Node, por ejemplo: AddTwoIntsClient.

🔹 Creación del cliente

  • En el constructor (__init__):

    • Importar la interfaz:
      from example_interfaces.srv import AddTwoInts

    • Crear el cliente de servicio:
      self.client = self.create_client(AddTwoInts, 'add_two_ints')

🔹 Método para enviar solicitudes

  • Crear un método como call_add_two_ints(self, a, b):

    1. Espera a que el servicio esté disponible con self.client.wait_for_service()

    2. Crea el objeto request con los valores a y b.

    3. Enviar la solicitud con self.client.call_async(request) que devuelve un future.

    4. Asociar un callback con future.add_done_callback().

🔹 Callback para manejar la respuesta

  • Crear un método como callback_call_add_two_ints(self, future, request):

    • Obtener la respuesta con future.result()

    • Mostrar en el log la operación completa: a + b = sum

    • Para poder acceder a los valores del request dentro del callback, se pasa el request como argumento adicional usando functools.partial.

🔹 Uso de functools.partial

  • Importar con:
    from functools import partial

  • Para pasar argumentos adicionales a un callback de un future, se debe usar partial, ya que add_done_callback() solo acepta un argumento (el future).

  • Ejemplo:
    future.add_done_callback(partial(self.callback_call_add_two_ints, request=request))

🔹 Ejecución desde el main

  • Crear el nodo y luego llamar al método call_add_two_ints() pasando diferentes valores.

  • Hacer que el nodo "spin" para permitir que los callbacks se ejecuten.

🔹 Envío de múltiples solicitudes

  • El mismo cliente puede enviar múltiples solicitudes consecutivas, por ejemplo:

    • call_add_two_ints(2, 7)

    • call_add_two_ints(1, 4)

    • call_add_two_ints(10, 20)

  • El servidor responderá a cada una.

🔹 Validaciones

  • El nodo debe estar corriendo (rclpy.spin(node)) para que las respuestas puedan ser recibidas.

  • Se debe registrar correctamente el nuevo ejecutable en setup.py.
