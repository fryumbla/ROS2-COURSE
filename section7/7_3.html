7.3. Práctica con parámetros en ROS 2 usando TurtleSim

Aplicarás tus conocimientos de parámetros en una práctica interactiva usando TurtleSim para modificar el comportamiento de la simulación.

📘 Notas de la clase

🔹 Iniciar TurtleSim

Ejecuta el nodo principal:

ros2 run turtlesim turtlesim_node

🔹 Listar nodos activos

ros2 node list

Resultado esperado:

  • /turtlesim

🔹 Ver parámetros del nodo

ros2 param list

Resultado esperado:

  • /turtlesim:

    • background_r

    • background_g

    • background_b

    • use_sim_time

    • parameter_events_queue_size

🔹 Obtener el valor de un parámetro

ros2 param get /turtlesim background_b

Resultado:

  • Muestra el tipo (Integer) y el valor (ej. 255)

Puedes repetir con background_r y background_g.

🔹 Cambiar valores de parámetros al lanzar el nodo

Usa --ros-args seguido de -p para asignar nuevos valores.

Ejemplo:

ros2 run turtlesim turtlesim_node --ros-args -p background_b:=0

Resultado: se elimina la componente azul del fondo → cambio de color.

Puedes modificar varios valores a la vez:

ros2 run turtlesim turtlesim_node --ros-args -p background_r:=200 -p background_g:=100 -p background_b:=50

🔹 ¿Por qué funciona esto?

  • Los parámetros están declarados con valores por defecto en el código de turtlesim.

  • Al usar --ros-args -p, se sobrescriben esos valores en tiempo de ejecución sin modificar el código fuente.

  • Es una manera muy práctica de personalizar nodos reutilizables.

🔹 Usar servicios para consultar parámetros

Listar servicios del nodo:

ros2 service list

Ejemplo de servicios relevantes:

  • /turtlesim/describe_parameters

  • /turtlesim/get_parameter_types

  • /turtlesim/list_parameters

Obtener tipo de un servicio:

ros2 service type /turtlesim/list_parameters

Resultado:
rcl_interfaces/srv/ListParameters

Ver contenido de la interfaz:

ros2 interface show rcl_interfaces/srv/ListParameters

🔹 Llamar a un servicio para listar parámetros

Comando:

ros2 service call /turtlesim/list_parameters rcl_interfaces/srv/ListParameters "{}"

Resultado:

  • Devuelve una lista con todos los nombres de parámetros declarados en el nodo /turtlesim

🔹 ¿Cuándo usar servicios en lugar de comandos CLI?

  • La CLI (ros2 param list, ros2 param get, etc.) es más fácil para pruebas interactivas.

  • Los servicios son útiles cuando quieres interactuar desde otro nodo programáticamente.

    • Ejemplo: un nodo cliente que obtiene parámetros de otro nodo.

  • No es algo común para principiantes, pero es bueno saber que es posible.

💾 Código trabajado / recursos

  • Nodo usado: turtlesim_node (ya implementado por ROS 2)

  • Herramientas utilizadas:

    • ros2 param list

    • ros2 param get

    • ros2 run ... --ros-args -p ...

    • ros2 service list

    • ros2 service call ...
