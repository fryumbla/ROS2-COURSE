7.1. Uso de parámetros en nodos ROS 2 con Python

Aprenderás cómo declarar, leer y modificar parámetros en nodos Python de ROS 2 durante la ejecución.

📘 Notas de la clase

🔹 Nodo base utilizado

  • El nodo reutilizado es number_publisher.py, que originalmente publica el número 2 cada 1 segundo.

  • El objetivo ahora es reemplazar esos valores fijos por parámetros configurables.

🔹 Declaración de parámetros

Dentro del constructor del nodo (__init__), antes de crear el publisher o el timer, se agregan las siguientes líneas:

  • Declarar el parámetro number (entero):

    self.declare_parameter("number", 2)

  • Declarar el parámetro timer_period (flotante):

    self.declare_parameter("timer_period", 1.0)

Esto:

  • Registra el parámetro dentro del nodo.

  • Establece el tipo de dato según el valor por defecto.

🔹 Obtención de valores de parámetros

Después de declarar, hay que obtener sus valores:

  • Obtener el valor del parámetro number:

    self.number = self.get_parameter("number").value

  • Obtener el valor del parámetro timer_period:

    self.timer_period = self.get_parameter("timer_period").value

Nota: .value es un atributo, no una función → no uses paréntesis

🔹 Uso en el nodo

  • self.number se usa como el valor a publicar.

  • self.timer_period se usa al crear el timer:

    self.create_timer(self.timer_period, self.timer_callback)

🔹 Ejecución con parámetros en tiempo de ejecución

Ejemplo 1: ejecutar el nodo con valores personalizados

ros2 run my_py_pkg number_publisher --ros-args -p number:=3 -p timer_period:=0.5

  • El número publicado será 3

  • Se publicará cada 0.5 segundos

Ejemplo 2: solo cambiar uno de los parámetros

ros2 run my_py_pkg number_publisher --ros-args -p number:=7

  • Se publicará 7 cada 1 segundo (valor por defecto del timer_period)

🔹 Comprobación de parámetros activos

Para ver los parámetros registrados en un nodo:

ros2 param list

Ejemplo de salida:

  • /number_publisher

    • number

    • timer_period

    • _parameter_event_publisher

    • use_sim_time

🔹 Validación de tipos

El tipo del parámetro está definido por el valor por defecto al declararlo. Ejemplos:

  • Si se declaró como entero:

    self.declare_parameter("number", 2)

    Entonces:

    • -p number:=5 → ✅

    • -p number:=3.5 → ❌ Error: se esperaba un entero

Mensaje de error típico:

Invalid parameter: parameter of type double, expected type integer

🔹 Resumen del uso de parámetros

  1. Declarar el parámetro con declare_parameter("nombre", valor_por_defecto)

  2. Obtener su valor con get_parameter("nombre").value

  3. Usarlo dentro de tu nodo

  4. En tiempo de ejecución, modificarlo con --ros-args -p name:=value
