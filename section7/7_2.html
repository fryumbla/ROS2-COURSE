7.2. Uso de parámetros en nodos ROS 2 con C++

Aprenderás cómo declarar, leer y modificar parámetros en nodos C++ de ROS 2 durante la ejecución.

📘 Notas de la clase

🔹 Nodo base reutilizado

  • Nodo: number_publisher.cpp

  • Publica el número 2 cada 1 segundo usando un publisher y un timer.

🔹 Declaración de parámetros

Dentro del constructor de la clase nodo:

  • Declarar el parámetro number (entero):
    this->declare_parameter("number", 2);

  • Declarar el parámetro timer_period (flotante):
    this->declare_parameter("timer_period", 1.0);

Esto:

  • Crea el parámetro dentro del nodo.

  • Establece el tipo de dato (según el valor por defecto).

🔹 Obtener el valor del parámetro

  • Obtener el parámetro number:
    number_ = this->get_parameter("number").as_int();

  • Obtener el parámetro timer_period:
    double timer_period = this->get_parameter("timer_period").as_double();

Nota: en C++ debes usar .as_int() o .as_double() para obtener el valor según el tipo declarado.

🔹 Uso del valor del parámetro

  • number_ se usa para publicar el número.

  • timer_period se usa al crear el timer:
    this->create_wall_timer(std::chrono::duration<double>(timer_period), callback);

🔹 Ejecución del nodo con parámetros en tiempo de ejecución

Ejemplo:

ros2 run my_cpp_pkg number_publisher --ros-args -p number:=20 -p timer_period:=3.4

  • Publicará el número 20 cada 3.4 segundos.

🔹 Comprobación de parámetros activos

Ver parámetros del nodo en ejecución:

ros2 param list

Ejemplo de salida esperada:

  • /number_publisher

    • number

    • timer_period

🔹 Comportamiento al omitir o escribir mal un parámetro

  • Si no se proporciona un parámetro, se usará su valor por defecto.

  • Si se proporciona un parámetro no declarado, será ignorado (no aparece en la lista de parámetros).

  • Si se proporciona un tipo incorrecto, ROS 2 mostrará un error de tipo.

Ejemplo de error:
Si declaras number como int y usas -p number:=3.5, obtendrás:
Invalid parameter: parameter of type double, expected type integer

🔹 Buenas prácticas

  • Declarar todos los parámetros necesarios al inicio del constructor.

  • Obtener y usar sus valores de forma segura (as_int(), as_double(), etc.).

  • Verificar los nombres y tipos al pasarlos en tiempo de ejecución.
