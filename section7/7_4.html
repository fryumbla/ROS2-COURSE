7.4. Uso de archivos YAML para configurar parámetros en ROS 2

Aprenderás a usar archivos YAML para configurar parámetros de forma centralizada y reutilizable en tus nodos ROS 2.

📘 Notas de la clase

🔹 ¿Por qué usar YAML para parámetros?

  • Evita escribir múltiples -p en la línea de comandos.

  • Permite mantener configuraciones reutilizables para distintos entornos.

  • Puedes definir parámetros para varios nodos dentro de un solo archivo.

🔹 Estructura de un archivo YAML

  1. Crea una carpeta para tus configuraciones si lo deseas (opcional):
    ~/yaml_params/

  2. Crea un archivo con extensión .yaml:
    number_params.yaml

  3. Contenido básico del archivo:

Nombre del nodo → ros__parameters → lista de parámetros:

Ejemplo:

number_publisher:
  ros__parameters:
    number: 5
    timer_period: 0.7

Reglas importantes:

  • Usa 2 espacios por nivel de indentación.

  • Escribe ros__parameters exactamente así (con doble guion bajo).

  • Los nombres deben coincidir con los nombres internos de los nodos y parámetros.

🔹 Ejecutar un nodo con archivo YAML

Usa la opción --ros-args seguida de --params-file:

ros2 run my_py_pkg number_publisher --ros-args --params-file ~/yaml_params/number_params.yaml

Puedes verificar el resultado con:

ros2 topic echo /number
ros2 topic hz /number

🔹 Configurar múltiples nodos en un solo YAML

Puedes definir varias configuraciones de nodos así:

number_publisher_1:
  ros__parameters:
    number: 5
    timer_period: 0.7

number_publisher_2:
  ros__parameters:
    number: 3
    timer_period: 1.0

🔹 Hacer coincidir el nombre del nodo

Para que un nodo use la configuración correcta, debe coincidir exactamente con el nombre definido en el YAML.

Ejecutar el primer nodo:

ros2 run my_py_pkg number_publisher --ros-args -r __node:=number_publisher_1 --params-file ~/yaml_params/number_params.yaml

Ejecutar el segundo nodo:

ros2 run my_py_pkg number_publisher --ros-args -r __node:=number_publisher_2 --params-file ~/yaml_params/number_params.yaml

🔹 ¿Qué sucede si el nombre no coincide?

  • Si el nodo no se llama como una de las entradas en el YAML:

    • Los parámetros no se aplican.

    • El nodo usará los valores por defecto definidos en el código.

    • ros2 param list no mostrará los parámetros desde el archivo.

🔹 Ventajas del uso de YAML

  • Puedes tener múltiples archivos YAML según el contexto:

    • Ejemplo: config_test.yaml, config_sim.yaml, config_real.yaml

  • Es compatible con ros2 launch (lo veremos más adelante).

  • Muy útil en simulación o pruebas automatizadas.

💾 Código trabajado / recursos

  • Nodo: number_publisher.py o number_publisher.cpp

  • Archivo de parámetros: ~/yaml_params/number_params.yaml

  • Comandos utilizados:

    • --ros-args

    • --params-file

    • -r __node:=...
