5.3. Crear un subscriber en Python con ROS 2

En esta clase aprenderás a crear un nodo en Python que actúe como suscriptor, recibiendo y procesando mensajes de un topic. Completarás el ciclo de comunicación pub-sub con Python.

📘 Notas de la clase

📂 Preparación

  • Ubícate en el directorio src del workspace de ROS 2.

  • Usa el mismo paquete Python, por ejemplo my_py_pkg.

  • Crea un nuevo archivo llamado smartphone.py.

  • Hazlo ejecutable: chmod +x smartphone.py

🧱 Estructura base del nodo

  • Usa la plantilla estándar de nodo ROS 2.

  • Clase Smartphone que hereda de Node.

  • Nombre del nodo: "smartphone" (evita redundancias como smartphone_node).

  • Método main() que inicializa y lanza el nodo.

📥 Crear un subscriber

  • Dentro del constructor __init__:

    • Usa self.create_subscription() con los siguientes argumentos:

      • Tipo de mensaje: String (igual al usado por el publisher)

      • Nombre del topic: "robot_news"

      • Función callback para manejar los mensajes recibidos

      • Tamaño de queue: 10

    • Código de importación: from example_interfaces.msg import String

📣 Crear la función callback

  • Método de clase llamado callback_robot_news(self, msg: String)

  • Accede al contenido con msg.data

  • Ejemplo:

    'console_scripts': [
        'smartphone = my_py_pkg.smartphone:main',
    ]

  • Esta función se ejecuta automáticamente cada vez que se recibe un mensaje.

🔧 Compilar e instalar

  • Compila con: colcon build --packages-select my_py_pkg --symlink-install

▶️ Ejecutar y probar

  1. En una terminal:
    ros2 run my_py_pkg smartphone
    Deberías ver:
    Smartphone has been started

  2. En otra terminal, ejecuta el publisher:
    ros2 run my_py_pkg robot_news_station

  3. Resultado esperado: la terminal del subscriber mostrará los mensajes recibidos, por ejemplo:
    Hi, this is C-3PO from the Robot News Station

  4. Al detener el publisher, el subscriber sigue activo pero no recibe más datos.

  5. Al reiniciar el publisher, el subscriber vuelve a mostrar los mensajes.

🧠 Conceptos clave repasados

  • Un subscriber:

    • No controla cuándo recibe mensajes

    • Reacciona a través de una función callback

  • El nombre del topic y el tipo de mensaje deben coincidir exactamente con los del publisher

  • El tamaño de queue actúa como buffer ante pérdidas de red o mensajes grandes

  • ROS 2 permite múltiples subscribers para un mismo topic

  • Tipo de mensaje: example_interfaces/msg/String

  • Nombre del topic: robot_news

  • Callback: callback_robot_news

  • Ejecutable registrado como: smartphone = my_py_pkg.smartphone:main
