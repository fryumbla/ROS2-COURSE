5.5. Crear un nodo suscriptor en C++ para ROS 2

En esta clase aprenderás a crear un nodo suscriptor en C++ que escuche mensajes de un topic. Completarás el ciclo de comunicación pub-sub con C++.

📘 Notas de la clase

📂 Estructura general

  • Se trabaja dentro del paquete de C++ ya creado (my_cpp_pkg).

  • Se crea un nuevo archivo fuente llamado smartphone.cpp.

🛠️ Creación del nodo suscriptor

  1. Archivo fuente:

    • Crear smartphone.cpp en el directorio src/ del paquete.

    • Utilizar una plantilla de nodo ROS 2 en C++ y renombrar el nodo como smartphone.

  2. Inclusión de la interfaz de mensaje:

    • Usar el mismo tipo de mensaje que el publicador:
      #include "example_interfaces/msg/string.hpp"

    • Verificar que example_interfaces esté declarado en package.xml y CMakeLists.txt.

  3. Callback del suscriptor:

    • Definir un método:
      void callback_robot_news(const std::shared_ptr<const example_interfaces::msg::String> msg)

    • Dentro del callback, imprimir el mensaje con RCLCPP_INFO:
      Usar msg->data.c_str() para convertir el mensaje a C-string.

  4. Declaración del suscriptor (atributo privado):

    • rclcpp::Subscription<example_interfaces::msg::String>::SharedPtr subscriber_;

  5. Inicialización del suscriptor en el constructor:

    • Usar create_subscription con:

      • Nombre del tópico: "robot_news"

      • Tamaño de la cola: 10

      • Callback: usar std::bind(&Smartphone::callback_robot_news, this, _1)

    • Para simplificar: agregar using namespace std::placeholders; al inicio del archivo.

  6. Mensaje de inicio del nodo:

    • Añadir:
      RCLCPP_INFO(this->get_logger(), "Smartphone node has been started.");

⚙️ Configuración del CMakeLists.txt

  • Añadir el nuevo ejecutable:
    add_executable(smartphone src/smartphone.cpp)

  • Especificar las dependencias:
    ament_target_dependencies(smartphone rclcpp example_interfaces)

  • Incluir el ejecutable en la instalación:
    install(TARGETS smartphone ...)

🚀 Compilar y ejecutar

  1. Compilar:
    colcon build --packages-select my_cpp_pkg

  2. Ejecutar el nodo suscriptor:
    . install/setup.bash
    ros2 run my_cpp_pkg smartphone

  3. Verificar el nodo:
    ros2 node list
    ros2 node info /smartphone

  4. Probar con publicadores:

    • Puedes usar cualquier publicador (C++ o Python) que publique en el tópico robot_news con el tipo std_msgs/String.

    • Al suscriptor no le importa quién publica, siempre que el tipo y el tópico coincidan.
