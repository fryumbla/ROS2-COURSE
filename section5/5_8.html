5.8. Monitoreo de tópicos con RQT Graph

En esta clase aprenderás a visualizar gráficamente todos los tópicos y sus conexiones utilizando rqt_graph. Esta herramienta es esencial para depurar y entender la arquitectura de comunicación de tu sistema ROS 2.

📘 Notas de la clase

🖼️ Iniciar el monitoreo con rqt_graph

  • Ejecutar:
    rqt_graph

  • En la interfaz visual, se mostrarán nodos (círculos) y tópicos (cuadrados).

  • Las flechas indican el flujo de datos entre publicadores y suscriptores.

🧪 Ejecutar nodos para visualizar actividad

  1. Publicador en C++ (robot_news_station):
    ros2 run my_cpp_pkg robot_news_station

  2. Suscriptor en Python (smartphone):
    ros2 run my_py_pkg smartphone

  3. Refrescar en rqt_graph:

    • Seleccionar la vista: Nodes/Topics (all)

    • Verás el tópico robot_news con un publicador y un suscriptor conectados

➕ Añadir múltiples publicadores y suscriptores

  • Nuevo publicador con nombre diferente:
    ros2 run my_py_pkg robot_news_station --ros-args -r __node:=station_two

  • Otro publicador (station_three):
    ros2 run my_py_pkg robot_news_station --ros-args -r __node:=station_three

  • Verificación:
    Todos los publicadores están conectados al mismo tópico robot_news
    Todos los suscriptores reciben los mensajes

  • Nuevo suscriptor (smartphone_two):
    ros2 run my_cpp_pkg smartphone --ros-args -r __node:=smartphone_two

🧭 Publicador con tópico diferente

  • Crear publicador con remapeo del tópico:
    ros2 run my_py_pkg robot_news_station --ros-args -r __node:=station_four -r robot_news:=my_news

  • Resultado:
    station_four publica en el tópico my_news, independiente del resto

🎯 Añadir suscriptor para my_news

  • Nuevo suscriptor (smartphone_three):
    ros2 run my_py_pkg smartphone --ros-args -r __node:=smartphone_three -r robot_news:=my_news

  • Verificación en rqt_graph:

    • smartphone_three solo está conectado al tópico my_news

    • Recibe solo los datos enviados por station_four

🧠 Análisis y buenas prácticas

  • rqt_graph permite detectar errores comunes:

    • Nombres de tópicos mal configurados

    • Nodos desconectados

    • Flujo de datos inesperado

  • Puedes crear sistemas con múltiples "canales" de comunicación usando diferentes tópicos

  • Renombrar nodos y tópicos en tiempo de ejecución permite:

    • Probar diferentes configuraciones sin tocar el código

    • Ejecutar múltiples instancias de un mismo nodo sin conflictos
