5.7. Renombrar nodos y tópicos en tiempo de ejecución con ROS 2

En esta clase aprenderás a utilizar el remapeo para renombrar nodos y tópicos en tiempo de ejecución. Esta técnica es fundamental para reutilizar código y crear sistemas modulares con ROS 2.

📘 Notas de la clase

🔁 Renombrar un nodo en tiempo de ejecución

  • Comando general:
    ros2 run <paquete> <ejecutable> --ros-args -r __node:=<nuevo_nombre>

  • Ejemplo:
    ros2 run my_py_pkg robot_news_station --ros-args -r __node:=my_station

  • Verificar:
    ros2 node list → debe aparecer /my_station

🔃 Renombrar un tópico en tiempo de ejecución

  • Sintaxis para remapear un tópico:
    --ros-args -r <nombre_original_tópico>:=<nuevo_nombre>

  • Ejemplo renombrando el tópico robot_news a abc:
    ros2 run my_py_pkg robot_news_station --ros-args -r __node:=my_station -r robot_news:=abc

  • Verificar:
    ros2 topic list → ahora verás /abc en lugar de /robot_news
    ros2 topic echo /abc → para visualizar los mensajes

🧩 ¿Qué pasa si no coinciden los nombres?

  • Si un publicador está usando /abc y un suscriptor /robot_news, no habrá comunicación.

  • Ambos deben coincidir en nombre y tipo de dato.

🛠️ Solución: Remapeo del suscriptor

  • Para hacer que un suscriptor escuche en /abc en lugar de /robot_news:
    ros2 run my_py_pkg smartphone --ros-args -r robot_news:=abc

  • Verifica que ahora el suscriptor reciba los datos correctamente.

⚠️ Consideraciones importantes

  • Solo se debe escribir --ros-args una vez, sin importar cuántos remapeos hagas.

  • Cada remapeo se especifica con -r seguido del patrón:
    nombre_antiguo:=nombre_nuevo

  • Esto funciona con nodos en Python y en C++, y no requiere modificar el código.
