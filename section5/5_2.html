5.2. Crear un publisher en Python con ROS 2

En esta clase aprenderás a crear un nodo en Python con ROS 2 que actúe como publisher, publicando mensajes de texto en un topic dos veces por segundo. Este es tu primer paso práctico para comunicar nodos mediante topics en ROS 2.

📘 Notas de la clase

📂 Preparación del entorno

  • Navega al workspace de ROS 2, dentro de src.

  • Usa un paquete Python ya existente, por ejemplo my_py_pkg.

  • Crea un nuevo archivo Python dentro del subdirectorio homónimo:
    robot_news_station.py

🔧 Hacer el archivo ejecutable

  • En terminal: chmod +x robot_news_station.py

🧱 Estructura base del nodo

  • Utiliza la plantilla estándar de nodo Python en ROS 2:

    • Importación de rclpy

    • Clase que hereda de Node

    • Método main() que inicializa y lanza el nodo

📢 Crear un publisher

  • Dentro del constructor __init__ de la clase:

    • Usa self.create_publisher() con:

      • Tipo de mensaje: String (de example_interfaces.msg)

      • Nombre del topic: robot_news

      • Tamaño de queue: 10

    • Código de ejemplo para importar el tipo: from example_interfaces.msg import String

📦 Añadir dependencia al archivo package.xml

  • Asegúrate de incluir: <depend>example_interfaces</depend>

⏱ Publicar mensajes periódicamente

  • Define una función publish_news() que:

    • Crea un mensaje de tipo String

    • Asigna contenido a msg.data

    • Usa self.publisher.publish(msg) para enviarlo

  • Crea un timer con self.create_timer(0.5, self.publish_news) para llamar a la función cada 0.5 segundos

🧪 Prueba de funcionalidad

  • Añade un log al iniciar el nodo: self.get_logger().info("Robot News Station has been started")
