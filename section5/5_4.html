5.4. Crear un nodo publicador en C++ para ROS 2

En esta clase aprenderás a crear un nodo publicador en C++ que publique mensajes en un topic. Implementarás la misma funcionalidad que en Python pero utilizando C++ y rclcpp.

📘 Notas de la clase

📂 Preparación del entorno

  • Ir al workspace: ~/ros2_ws/src/

  • Entrar al paquete en C++ (por ejemplo my_cpp_pkg)

  • Crear un nuevo archivo fuente: robot_news_station.cpp

🔧 Configuración inicial

  • Utilizar una plantilla de nodo ROS 2 y renombrar:

    • Clase: RobotNewsStation

    • Nombre del nodo: "robot_news_station"

  • El archivo debe tener:

    • Inclusión de rclcpp/rclcpp.hpp

    • Creación del nodo con rclcpp::Node

    • Función main que ejecuta el spin

📥 Inclusión del tipo de mensaje

  • Incluir el mensaje String desde el paquete example_interfaces:
    #include "example_interfaces/msg/string.hpp"

  • Verificar dependencias:

    • En package.xml, agregar:
      <depend>example_interfaces</depend>

    • En CMakeLists.txt:
      Agregar find_package(example_interfaces REQUIRED)
      Incluir example_interfaces en ament_target_dependencies

📤 Crear el publicador

  1. Declarar como atributo privado:
    rclcpp::Publisher<example_interfaces::msg::String>::SharedPtr publisher_;

  2. Inicializar en el constructor:
    publisher_ = this->create_publisher<example_interfaces::msg::String>("robot_news", 10);

    • "robot_news" es el nombre del tópico

    • 10 es el tamaño de la cola (buffer)

✉️ Publicar mensajes

  1. Crear método privado publish_news:
    void publish_news() {
    auto msg = example_interfaces::msg::String();
    std::string robot_name = "R2D2";
    msg.data = "Hi, this is " + robot_name + " from the robot news station.";
    publisher_->publish(msg);
    }

  2. Crear timer_ para llamar periódicamente al método:

    • Declarar en la clase:
      rclcpp::TimerBase::SharedPtr timer_;

    • Inicializar en el constructor:
      using namespace std::chrono_literals;
      timer_ = this->create_wall_timer(500ms, std::bind(&RobotNewsStation::publish_news, this));

  3. Log de confirmación al iniciar el nodo:
    RCLCPP_INFO(this->get_logger(), "Robot News Station has been started.");

⚙️ Configuración del CMakeLists.txt

  • Añadir el nuevo ejecutable:
    add_executable(robot_news_station src/robot_news_station.cpp)

  • Declarar dependencias:
    ament_target_dependencies(robot_news_station rclcpp example_interfaces)

  • Instalar el ejecutable:
    install(TARGETS robot_news_station ...)

🚀 Compilar y ejecutar

  1. Compilar el paquete:
    colcon build --packages-select my_cpp_pkg

  2. Ejecutar el nodo:
    . install/setup.bash
    ros2 run my_cpp_pkg robot_news_station

  3. Verificar funcionamiento:
    ros2 node list
    ros2 node info /robot_news_station
    ros2 topic echo /robot_news

🔁 Prueba cruzada con Python

  • Ejecutar un suscriptor hecho en Python (smartphone) que escuche en el mismo tópico /robot_news

  • Comprobar que recibe correctamente los mensajes del publicador en C++

  • Confirmación: ROS 2 es agnóstico al lenguaje — los nodos en C++ y Python pueden comunicarse si usan el mismo tópico y tipo de dato
