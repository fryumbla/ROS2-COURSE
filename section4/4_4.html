4.4. Visualizar nodos con rqt_graph en ROS 2

En esta clase aprenderás a utilizar rqt_graph, una herramienta gráfica de ROS 2 que te permite visualizar en tiempo real los nodos y sus conexiones. Es ideal para entender y depurar la estructura de tu aplicación ROS.

📘 Notas de la clase

1. ¿Qué es rqt?

  • rqt es una interfaz gráfica modular para ROS.

  • Permite realizar introspección, configurar parámetros, visualizar logs, probar servicios y mucho más.

  • Es equivalente al uso de comandos ros2, pero con GUI.

2. Cómo iniciar rqt

  • Abre una terminal y ejecuta:

rqt

  • Carga varios plugins útiles. El que nos interesa por ahora es:

📌 rqt_graph → Visualiza todos los nodos activos y sus conexiones.

3. Iniciar rqt_graph directamente

Si solo quieres abrir la vista del grafo:

rqt_graph

Inicialmente estará vacío si no hay nodos activos.

4. Ver nodos activos en tiempo real

  • Ejecuta tus nodos como siempre, por ejemplo:

ros2 run my_py_pkg py_node  
ros2 run my_cpp_pkg cpp_node

  • Luego vuelve a rqt_graph y pulsa el botón de 🔁 refrescar.

  • Aparecerán los nodos activos (ej: /py_test, /cpp_test).

5. Mostrar/Ocultar nodos del sistema

  • En rqt_graph, puedes desmarcar la opción "Debug" para ocultar nodos internos como:

    • /ros2cli_daemon (proceso de línea de comandos)

    • /rqt_gui_py_node_xxx (el nodo del propio rqt)

  • Puedes volver a activarlos cuando lo necesites.

6. Visualizar conexiones entre nodos

  • rqt_graph muestra las flechas entre nodos conectados mediante topics.

  • Por ahora verás flechas hacia /rosout, ya que todos los nodos publican logs allí.

  • Más adelante, cuando tus nodos usen publish/subscribe, verás conexiones adicionales.

7. ¿Por qué usar rqt_graph?

✅ Visualización clara y global del estado del sistema
✅ Detección fácil de errores de nombre o comunicación
✅ Muy útil al trabajar con muchos nodos o sistemas distribuidos
✅ Alternativa gráfica a ros2 node list e info

💾 Código trabajado / recursos

Ejecutables visualizados: py_node y cpp_node  
Herramienta utilizada: rqt_graph (GUI)
