4.3. Opción --symlink-install para desarrollo rápido de nodos Python en ROS 2

En esta clase aprenderás cómo usar la opción --symlink-install al compilar paquetes Python en ROS 2, para acelerar el ciclo de desarrollo sin necesidad de recompilar cada vez que haces cambios en el código.

📘 Notas de la clase

1. Recordatorio: Dónde compilar

  • Siempre compila desde la raíz del workspace:
    ~/ws

  • No compiles desde src, desde el paquete, ni desde ningún otro directorio.

2. Comando básico para compilar

colcon build

  • Compila todos los paquetes encontrados en ~/ws/src.

Para compilar solo un paquete:

colcon build --packages-select my_py_pkg

3. Nueva opción: --symlink-install (solo para paquetes Python)

Para permitir ejecutar directamente archivos .py desde src sin volver a compilar cada vez:

colcon build --packages-select my_py_pkg --symlink-install

¿Qué hace esta opción?

  • Crea enlaces simbólicos desde install/ hacia tu carpeta src/

  • Permite que ros2 run ejecute directamente tu script .py actualizado

  • Ya no es necesario compilar de nuevo tras cada cambio en el archivo .py

4. Flujo de trabajo con --symlink-install

  • 1️⃣ Compilas una vez con --symlink-install

  • 2️⃣ Haces cambios en tu archivo Python

  • 3️⃣ Guardas el archivo

  • 4️⃣ Ejecutas de nuevo con ros2 run → ¡El cambio ya se refleja!

5. Ejemplo de uso

ros2 run my_py_pkg py_node

Luego, editas my_first_node.py, guardas, y al ejecutar otra vez:

ros2 run my_py_pkg py_node

… ya verás los cambios sin necesidad de compilar de nuevo.

6. Advertencias importantes

⚠️ Esta opción es solo para desarrollo, no para producción.

⚠️ Solo funciona con paquetes Python.

⚠️ Puede ocasionar errores o comportamientos inesperados. Si algo falla:

  • Vuelve a compilar sin --symlink-install

  • Usa el flujo estándar: build → source → run

7. Resumen de uso

✅ Ideal para iteraciones rápidas mientras desarrollas nodos en Python
❌ No usar con paquetes C++
❌ No recomendado para despliegue o producción

💾 Código trabajado / recursos

Nodo Python: ~/ws/src/my_py_pkg/my_py_pkg/my_first_node.py  
Build con: colcon build --packages-select my_py_pkg --symlink-install
