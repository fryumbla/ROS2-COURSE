4.1. Ejecutar e inspeccionar nodos con la línea de comandos de ROS 2

En esta clase aprenderás a ejecutar tus nodos desde la terminal usando ros2 run y a inspeccionar los nodos activos con herramientas como ros2 node list e info. También verás la importancia de mantener nombres únicos para cada nodo.

📘 Notas de la clase

1. Asegurar el entorno correctamente "sourceado"

Verifica que tu archivo ~/.bashrc contenga al final las siguientes líneas:

  • Para ROS 2:
    source /opt/ros/<distro>/setup.bash

  • Para tu workspace:
    source ~/ws/install/setup.bash

Esto garantiza que al abrir una terminal nueva puedas ejecutar comandos ros2 y acceder a los ejecutables instalados en tu workspace.

2. Usar ros2 run para ejecutar nodos

Sintaxis:

ros2 run <paquete> <ejecutable>

Ejemplos:

  • Nodo Python:
    ros2 run my_py_pkg py_node

  • Nodo C++:
    ros2 run my_cpp_pkg cpp_node

Si olvidaste el nombre del ejecutable, puedes usar la tecla Tab dos veces después del nombre del paquete para que ROS 2 lo autocomplete.

3. Problemas comunes: "package not found"

Este error suele aparecer cuando:

  • No has hecho source ~/ws/install/setup.bash

  • El ~/.bashrc no incluye el source del workspace

  • Has construido el paquete en una terminal, pero no has hecho source en esa misma terminal

✅ Solución:

  • Ejecuta source ~/.bashrc o abre una terminal nueva

4. Ver la lista de nodos activos

Comando:

ros2 node list

Muestra los nombres de los nodos actualmente ejecutándose.

Ejemplo de salida:

  • /py_test

  • /cpp_test

5. Obtener información detallada de un nodo

Comando:

ros2 node info <nombre_del_nodo>

Ejemplo:

ros2 node info /cpp_test

Salida esperada:

  • Publicadores y subscriptores

  • Servicios utilizados por el nodo

  • Información útil para debugging o integración

6. Obtener ayuda de cualquier comando ROS 2

Agrega -h para ver la ayuda de un comando:

  • ros2 run -h

  • ros2 node -h

7. Reglas importantes

  • No puedes tener dos nodos activos con el mismo nombre.
    Si ejecutas dos nodos con el mismo nombre, ROS 2 mostrará un error o reemplazará el nodo anterior.

8. Herramientas de terminal recomendadas

  • El instructor usa Terminator para dividir y trabajar con varias terminales al mismo tiempo.
    Puedes instalarlo para facilitar tu flujo de trabajo.
