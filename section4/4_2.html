4.2. Ejecutar múltiples instancias de un nodo usando remapeo de nombres en ROS 2

En esta clase aprenderás cómo ejecutar varias instancias de un mismo nodo en ROS 2, asignándoles diferentes nombres al momento de lanzarlos. Esto te permitirá usar el mismo código para múltiples sensores o componentes, sin conflictos de nombres.

Documentación
https://docs.ros.org/en/jazzy/How-To-Guides/Node-arguments.html


📘 Notas de la clase

1. ¿Por qué cambiar el nombre de un nodo?

  • En ROS 2 no pueden existir dos nodos con el mismo nombre activos al mismo tiempo.

  • Si intentas lanzar el mismo nodo dos veces, verás una advertencia como:

    "There are nodes in the graph that share an exact name, which can have unintended side effects."

  • Esto puede provocar errores inesperados.

2. Solución: renombrar nodos al ejecutarlos

Puedes asignar un nuevo nombre a un nodo en tiempo de ejecución, sin modificar el código fuente.

3. Sintaxis para cambiar el nombre del nodo

ros2 run <paquete> <ejecutable> --ros-args -r __node:=<nuevo_nombre>

📌 Ejemplo:

ros2 run my_py_pkg py_node --ros-args -r __node:=abc

Esto iniciará el nodo con el nombre abc en lugar del nombre definido originalmente (py_test en el código).

4. Ejecutar múltiples instancias

Ejemplo típico: Tienes un nodo para un sensor de temperatura y cinco sensores en total.

Puedes iniciar múltiples nodos así:

  • ros2 run my_py_pkg py_node --ros-args -r __node:=temperature_sensor_1

  • ros2 run my_py_pkg py_node --ros-args -r __node:=temperature_sensor_2

  • ...

Cada uno será una instancia independiente con un nombre único.

5. Variaciones de la opción -r

La opción -r es una forma abreviada de --remap, ambas funcionan:

  • -r __node:=nombre

  • --remap __node:=nombre

6. Ventajas del renombramiento en tiempo de ejecución

✅ No es necesario modificar el código fuente
✅ No es necesario recompilar ni reinstalar
✅ Ideal para nodos reutilizables y dinámicos
✅ Útil para sensores, múltiples robots, simulaciones, etc.

7. Recordatorio: ROS arguments (--ros-args)

  • Se usa para pasar configuraciones adicionales a los nodos

  • Solo se coloca una vez por comando

  • A partir de --ros-args, puedes añadir:

    • Remapeos de nombre (-r)

    • Parámetros (--param)

    • Remapeos de topics y servicios (lo veremos más adelante)
