4.5. Introducción a TurtleSim y ejecución de nodos en ROS 2

En esta clase aprenderás a iniciar y controlar una simulación 2D básica usando el paquete turtlesim en ROS 2, aplicando los conocimientos previos sobre ejecución de nodos y comunicación entre ellos.

📝 Documentación
https://docs.ros.org/en/jazzy/Tutorials/Beginner-CLI-Tools/Introducing-Turtlesim/Introducing-Turtlesim.html


📘 Notas de la clase

¿Qué es TurtleSim?

  • turtlesim es un paquete de simulación 2D que permite controlar una "tortuga" en un entorno gráfico.

  • Es ideal para practicar conceptos de ROS 2 de forma visual y sencilla.

Cómo ejecutar el nodo de TurtleSim

  1. Abre una terminal.

  2. Ejecuta el nodo con: ros2 run turtlesim turtlesim_node

  3. Si recibes un error indicando que el paquete no está instalado: sudo apt install ros-<distro>-turtlesim (Reemplaza <distro> con tu distribución actual de ROS, por ejemplo humble o foxy).

  4. Al ejecutarlo correctamente, se abrirá una ventana gráfica con la tortuga.

Controlar la tortuga con el teclado

  1. Abre una segunda terminal.

  2. Ejecuta el siguiente comando para iniciar el nodo que permite el control por teclado:
    ros2 run turtlesim turtle_teleop_key

  3. Este nodo lee las teclas del teclado (flechas) y envía comandos de movimiento a la tortuga.

  4. Asegúrate de tener seleccionada la terminal activa antes de presionar las teclas.

Visualizar la comunicación entre nodos

  1. Abre una tercera terminal.

  2. Ejecuta el comando:
    ros2 run rqt_graph rqt_graph

  3. En la ventana que aparece puedes:

    • Cambiar la vista a "Nodes Only" o "Nodes/Topics".

    • Ver cómo el nodo turtle_teleop_key se comunica con turtlesim_node a través del tópico /turtle1/cmd_vel.

Renombrar nodos usando argumentos de ROS

  • Puedes asignar un nombre personalizado al nodo al momento de ejecutarlo:
    ros2 run turtlesim turtlesim_node --ros-args -r __node:=mi_tortuga

  • En el gráfico de nodos, verás el nuevo nombre reflejado.

Resumen de conceptos clave

  • Uso del comando ros2 run para iniciar nodos.

  • Interacción entre nodos mediante tópicos.

  • Visualización de la arquitectura del sistema con rqt_graph.

  • Argumentos de línea de comandos para remapeo y personalización.

💾 Código trabajado / recursos

  • Ejecutar nodo principal:
    ros2 run turtlesim turtlesim_node

  • Ejecutar nodo de teleoperación:
    ros2 run turtlesim turtle_teleop_key

  • Ejecutar visualizador gráfico de nodos:
    ros2 run rqt_graph rqt_graph

  • Instalar paquete (si es necesario):
    sudo apt install ros-<distro>-turtlesim
