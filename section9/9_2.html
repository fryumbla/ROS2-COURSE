9.2. Entender el árbol TF y las relaciones entre transformaciones

Comprenderás cómo funciona el árbol de transformaciones (TF tree) en ROS 2 y cómo se relacionan las diferentes transformaciones entre componentes del robot.

📘 Notas de la clase

🌐 ¿Qué es el árbol TF?

  • Es la estructura que describe cómo están conectados espacialmente los distintos frames (o partes) del robot.

  • Cada link está conectado mediante un TF que indica:

    • Su posición relativa

    • Su orientación

    • Su tipo de movimiento (fijo o articulado)

🔁 Relaciones entre TF

  • Si el right_leg es hijo del base_link, cualquier movimiento del base_link afecta al right_leg.

  • Es una estructura jerárquica: si mueves un nodo padre, todos sus hijos se mueven con él.

  • Similar a cómo tu codo afecta el movimiento de tu mano.

📡 Visualizar el árbol TF en RViz

  1. En RViz:

    • Activa el plugin TF

    • Desactiva el RobotModel si quieres ver solo los ejes

    • Puedes ver flechas entre frames indicando la relación padre-hijo

  2. Activar solo ciertos frames:

    • Por ejemplo: activa base_link y right_leg

    • Verás la flecha que los conecta

🧪 Observar comportamiento dinámico

  1. Mueve el gripper_extension con el panel de Joint State Publisher

  2. Verás que no solo se mueve el gripper, sino también sus elementos hijos

  3. Esto confirma que el árbol TF propaga los cambios de movimiento desde un nodo padre

📄 Ver el tópico /tf manualmente

  1. Abre una terminal nueva:

ros2 topic list

  1. Verifica que existe /tf:

ros2 topic echo /tf

  1. Verás mensajes con transformaciones, como:

  • frame_id: gripper_pole

  • child_frame_id: right_gripper

  • translation: x, y, z

  • rotation: quaternion

📌 Cada mensaje indica cómo ir de un frame a otro (posición y orientación en el espacio)

📤 Visualizar el árbol TF completo como imagen (PDF)

  1. Instala la herramienta si no la tienes:

sudo apt install ros-humble-tf2-tools

  1. Ejecuta el siguiente comando:

ros2 run tf2_tools view_frames

  1. Este comando:

    • Escucha /tf durante 5 segundos

    • Crea dos archivos: .pdf y .dot en el directorio actual

  2. Abre el archivo generado:

evince frames.pdf (o tu visor de PDF preferido)

✅ El PDF muestra el árbol completo: cada nodo representa un link, y cada flecha representa una transformación

🌍 Uso con múltiples robots

  • Un TF tree puede manejar múltiples robots.

  • Ejemplo:
    world
     ├── base_link_1
     ├── base_link_2
     └── base_link_3
