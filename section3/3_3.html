3.3. Creación de un paquete C++ en ROS 2

En esta clase aprenderás cómo crear un paquete C++ en ROS 2, utilizando el sistema de construcción ament_cmake. Verás cómo organizarlo dentro del workspace y compilarlo correctamente.

📘 Notas de la clase

1. Ubicación del paquete
Todos los paquetes deben crearse dentro del directorio src del workspace:
~/ws/src

2. Comando para crear un paquete C++
Desde ~/ws/src, ejecuta:

ros2 pkg create my_cpp_pkg --build-type ament_cmake --dependencies rclcpp

  • my_cpp_pkg es el nombre del paquete (usa guiones bajos si es necesario).

  • --build-type ament_cmake especifica que el paquete es en C++.

  • --dependencies rclcpp agrega la dependencia a la librería C++ de ROS 2.

Puedes agregar más dependencias separadas por espacios si lo necesitas.

3. Estructura generada del paquete

Al ejecutar el comando, se generará esta estructura:

  • my_cpp_pkg/

    • include/my_cpp_pkg/ → para archivos .hpp o .h

    • src/ → para archivos fuente .cpp

    • CMakeLists.txt → reglas de compilación

    • package.xml → metadatos y dependencias del paquete

4. Archivos clave

  • package.xml
    Obligatorio en todo paquete ROS 2. Contiene nombre, versión, descripción, licencia, autor y dependencias (como rclcpp).
    Puedes agregar más dependencias con etiquetas <depend>.

  • CMakeLists.txt
    Define cómo se compilarán los archivos C++.
    Aún no lo modificamos, se hará cuando se cree el primer nodo.

5. Apertura del proyecto en Visual Studio Code
Desde ~/ws/src, abre el editor con:

code .

Esto permite que VS Code detecte correctamente las rutas de inclusión y facilite el autocompletado.

6. Compilar el paquete

Nunca compiles desde ~/ws/src. Debes compilar siempre desde la raíz del workspace ~/ws.

Para compilar todo el workspace:

colcon build

7. Compilar solo un paquete específico (opcional)

colcon build --packages-select my_cpp_pkg

Esto es útil cuando hay múltiples paquetes y quieres compilar solo uno.

8. Error común: compilar en la carpeta equivocada
Si ejecutas colcon build dentro de ~/ws/src, se crearán carpetas build/, install/ y log/ allí.
Esto puede causar conflictos o errores.

Para solucionarlo:

rm -rf build install log

Luego vuelve a compilar correctamente desde ~/ws.
