3.5. Crear, instalar y ejecutar un nodo Python básico en ROS 2

En esta clase aprenderás a crear un nodo básico en Python dentro de tu paquete ROS 2, instalarlo y ejecutarlo correctamente desde la terminal usando ros2 run.

📘 Notas de la clase

1. Crear el archivo del nodo
Desde ~/ws/src/my_py_pkg/my_py_pkg/, crea el archivo:

touch my_first_node.py

Este archivo será donde escribas el código del nodo. Asegúrate de trabajar dentro del directorio que tiene el mismo nombre que tu paquete (my_py_pkg).

2. Estructura mínima del código de un nodo
Contenido básico del archivo my_first_node.py:

  • Línea shebang: #!/usr/bin/env python3

  • Importaciones:

    • import rclpy

    • from rclpy.node import Node

  • Función principal (main(args=None)) que:

    • Inicializa ROS 2 (rclpy.init)

    • Crea un nodo (node = Node('py_test'))

    • Imprime un mensaje en consola (node.get_logger().info('Hello world'))

    • Hace un spin del nodo (rclpy.spin(node))

    • Finaliza con rclpy.shutdown()

3. Ejecutar directamente el nodo (opcional)
Haz el archivo ejecutable:

chmod +x my_first_node.py

Y luego ejecútalo con:

./my_first_node.py

4. Instalar el nodo como ejecutable en ROS 2
Para poder ejecutar el nodo con ros2 run, edita el archivo setup.py ubicado en ~/ws/src/my_py_pkg/:

Dentro del bloque entry_points={ 'console_scripts': [...] }, añade:

"py_node = my_py_pkg.my_first_node:main"

Importante:

  • "py_node" será el nombre del ejecutable.

  • my_py_pkg.my_first_node es el archivo del nodo (sin .py).

  • main es el nombre de la función a ejecutar.

Guarda los cambios.

5. Compilar el paquete
Desde el workspace ~/ws, ejecuta:

colcon build --packages-select my_py_pkg

Luego, refresca el entorno:

source ~/.bashrc

6. Ejecutar el nodo instalado con ros2 run

ros2 run my_py_pkg py_node

  • my_py_pkg: nombre del paquete

  • py_node: nombre del ejecutable definido en setup.py

7. Diferencias entre nombres
Es importante entender que:

  • Nombre del archivo: my_first_node.py

  • Nombre del nodo: 'py_test' (definido dentro del código)

  • Nombre del ejecutable: py_node (definido en setup.py)

Estos tres nombres pueden ser iguales o diferentes, pero tienen propósitos distintos.

8. Mantener el nodo activo con spin()
La función rclpy.spin(node) mantiene el nodo ejecutándose hasta que se interrumpa manualmente (por ejemplo, con Ctrl+C).
