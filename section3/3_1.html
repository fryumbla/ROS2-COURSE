3.1. Creación y configuración del workspace en ROS 2

En esta clase aprenderás cómo crear un workspace en ROS 2, configurarlo correctamente y dejarlo listo para comenzar a desarrollar paquetes y ejecutar nodos en tu aplicación.

📘 Notas de la clase

1. ¿Qué es un workspace en ROS 2?
Un workspace es simplemente un directorio donde desarrollarás y compilarás tu aplicación en ROS 2. Es el entorno de trabajo para tus paquetes, nodos y scripts.

2. Creación del workspace paso a paso

  • Asegúrate de estar en tu directorio home:
    cd

  • Crea una carpeta para el workspace, por ejemplo:
    mkdir ws

  • Ingresa al nuevo directorio:
    cd ws

  • Dentro de este workspace, crea un subdirectorio llamado src (source):
    mkdir src

3. Estructura inicial del workspace

ws/
└── src/

4. Compilar el workspace por primera vez

  • Desde el directorio raíz del workspace (ws), ejecuta:
    colcon build

  • Esto genera tres nuevas carpetas automáticamente:
    build/, install/, y log/

5. Si el comando colcon no está disponible:
Asegúrate de haber instalado la herramienta:
sudo apt install python3-colcon-common-extensions

6. Activar el entorno del workspace

  • Luego de compilar, ve al directorio install/ y verifica que existe el script setup.bash

  • Para acceder a las funcionalidades del workspace, se debe "sourcear" el archivo:
    source install/setup.bash

7. Automatizar el sourcing del workspace

  • Edita el archivo .bashrc:
    nano ~/.bashrc

  • Añade la siguiente línea después del sourcing de ROS global (asegúrate del orden):
    source ~/ws/install/setup.bash

  • Guarda el archivo y cierra.

  • Ahora, cada nueva terminal tendrá el workspace activo automáticamente.

8. Resultado final
El entorno está listo para comenzar a crear paquetes ROS 2 dentro del directorio src.
