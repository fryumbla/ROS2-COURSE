3.7. Crear, compilar e instalar un nodo C++ en ROS 2

En esta clase aprenderás a crear un nodo básico en C++ utilizando la estructura de un paquete ROS 2, compilarlo, instalarlo y ejecutarlo desde la terminal con ros2 run.

📘 Notas de la clase

1. Crear el archivo del nodo C++
Desde el directorio del paquete:

touch ~/ws/src/my_cpp_pkg/src/my_first_node.cpp

Este archivo contendrá el código fuente del nodo.

2. Estructura mínima del nodo en C++

  • Incluir la cabecera principal de ROS 2 para C++:

#include "rclcpp/rclcpp.hpp"

  • Crear la función main(int argc, char **argv)

    • Inicializar ROS 2: rclcpp::init(argc, argv);

    • Crear un nodo:
      auto node = std::make_sharedrclcpp::Node("cpp_test");

    • Imprimir un mensaje:
      RCLCPP_INFO(node->get_logger(), "Hello world");

    • Finalizar: rclcpp::shutdown();

    • Retornar 0

3. Usar rclcpp::spin(node) para mantener el nodo activo
Agregar antes de shutdown:

rclcpp::spin(node);

Esto mantiene el nodo vivo hasta que se presione Ctrl+C.

4. Configurar CMakeLists.txt

Modificar ~/ws/src/my_cpp_pkg/CMakeLists.txt:

  • Después de find_package(...), agregar:

add_executable(cpp_node src/my_first_node.cpp)

ament_target_dependencies(cpp_node rclcpp)

  • Agregar al final (antes de ament_package()):

install(TARGETS
cpp_node
DESTINATION lib/${PROJECT_NAME}
)

5. Compilar el paquete

Desde el workspace ~/ws:

colcon build --packages-select my_cpp_pkg

6. Verificar errores comunes

  • Si olvidas pasar el nombre del nodo al constructor:
    std::make_shared<rclcpp::Node>("cpp_test")
    El build mostrará un error (típico en C++).

  • Siempre guarda los archivos antes de compilar.

7. Ejecutar el nodo C++

  • Refrescar el entorno:
    source install/setup.bash
    o source ~/.bashrc si ya lo tienes configurado

  • Ejecutar con:

ros2 run my_cpp_pkg cpp_node

8. Diferencias entre nombres importantes

  • Nombre del archivo: my_first_node.cpp

  • Nombre del nodo: "cpp_test" (definido en el constructor)

  • Nombre del ejecutable: cpp_node (definido en CMakeLists.txt)

Estos nombres pueden coincidir o no, pero cumplen funciones distintas.

9. Resumen del flujo de trabajo

Cada vez que edites el código:

  • Build: colcon build --packages-select my_cpp_pkg

  • Source: source ~/.bashrc

  • Run: ros2 run my_cpp_pkg cpp_node
