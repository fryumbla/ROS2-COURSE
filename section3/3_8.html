3.8. Nodo C++ con Programación Orientada a Objetos (OOP) y temporizador en ROS 2

En esta clase mejorarás tu nodo C++ aplicando programación orientada a objetos (OOP), y aprenderás cómo usar un temporizador para ejecutar acciones periódicas dentro del nodo.

📘 Notas de la clase

1. Estructura OOP de un nodo C++

Se define una clase propia que hereda de rclcpp::Node. Esta es la forma recomendada de escribir nodos en C++ en ROS 2, ya que facilita la organización y escalabilidad del código.

class MyNode : public rclcpp::Node {
  // ...
};
  • En el constructor se llama al constructor de la clase padre con el nombre del nodo:
    Node("cpp_test")

2. Funcionalidad básica del nodo

En el constructor:

  • Se imprime un log con:
    RCLCPP_INFO(this->get_logger(), "Hello world");

  • Se puede usar this-> para acceder a métodos de Node, aunque no es obligatorio.

3. Mantener el nodo activo con spin

Desde main():

  • Crear un shared_ptr del nodo:
    auto node = std::make_shared<MyNode>();

  • Iniciar ROS 2 con: rclcpp::init(argc, argv);

  • Ejecutar spin: rclcpp::spin(node);

  • Finalizar con rclcpp::shutdown();

4. Añadir un temporizador con callback

  • Crear un atributo privado para el temporizador:

    rclcpp::TimerBase::SharedPtr timer_;

  • Definir una función callback:

    void timer_callback();

  • En el constructor, registrar el temporizador:

    timer_ = this->create_wall_timer(
  std::chrono::seconds(1),
  std::bind(&MyNode::timer_callback, this)
);

    Esto ejecutará la función timer_callback cada segundo mientras el nodo esté vivo.

5. Añadir un contador

  • En los atributos privados, definir:
    int counter_ = 0;

  • En el callback:
    RCLCPP_INFO(this->get_logger(), "Hello %d", counter_);
    counter_++;

6. Compilar y ejecutar

🛠 Build
Desde ~/ws:

colcon build --packages-select my_cpp_pkg

🔁 Source
Actualizar el entorno:

source ~/.bashrc
o
source install/setup.bash

▶️ Run

ros2 run my_cpp_pkg cpp_node

7. Recapitulación del flujo

  • Se ejecuta el main(), que crea el nodo y lo mantiene vivo con spin().

  • El constructor del nodo se ejecuta automáticamente.

  • Se registra el temporizador.

  • El spin() permite ejecutar el callback periódicamente.

  • Al presionar Ctrl+C, se detiene el nodo y se llama a shutdown().

💡 Nota: No es necesario modificar CMakeLists.txt si ya está configurado de clases anteriores. Solo asegúrate de guardar todos los archivos antes de compilar.
