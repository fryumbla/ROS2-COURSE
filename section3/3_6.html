3.6. Nodo Python con Programación Orientada a Objetos (OOP) y temporizador en ROS 2

En esta clase mejorarás tu nodo Python aplicando programación orientada a objetos (OOP), y aprenderás cómo usar un temporizador para ejecutar acciones periódicas dentro del nodo.

📘 Notas de la clase

1. Uso de programación orientada a objetos (OOP) en nodos ROS 2
ROS 2 recomienda estructurar los nodos utilizando clases, ya que facilita la escalabilidad y el mantenimiento del código.

Estructura básica de un nodo con OOP:

  • Se define una clase que hereda de Node, por ejemplo:
    class MyNode(Node):

  • En el constructor (__init__), se llama al constructor del padre:
    super().__init__('py_test')

  • Se pueden utilizar funcionalidades de ROS 2 como el logger:
    self.get_logger().info('Hello world')

2. Modificación del archivo del nodo
El archivo my_first_node.py queda ahora estructurado como:

  • Clase MyNode que hereda de Node

  • Constructor con super().__init__('py_test')

  • Dentro del main, se crea una instancia:
    node = MyNode()

  • Se mantiene el patrón:

    • rclpy.init()

    • rclpy.spin(node)

    • rclpy.shutdown()

3. Añadir un temporizador al nodo
El temporizador permite ejecutar una función periódicamente mientras el nodo está activo. Se usa la función create_timer.

Pasos para agregar un temporizador:

  • Definir un método en la clase, por ejemplo timer_callback(self)

  • Usar dentro del constructor:

    self.create_timer(1.0, self.timer_callback)

    • 1.0: intervalo en segundos (1 segundo)

    • self.timer_callback: función a ejecutar periódicamente

4. Contador dentro del temporizador (opcional)

  • Se puede definir un contador como atributo de la clase:
    self.counter_ = 0

  • En el timer_callback, incrementar el contador y mostrarlo:

    self.get_logger().info('Hello ' + str(self.counter_))
    self.counter_ += 1

5. Compilación y ejecución

Pasos a seguir cada vez que modificas el nodo:

  • Build:
    colcon build --packages-select my_py_pkg

  • Source:
    source ~/.bashrc
    (o directamente: source install/setup.bash desde ~/ws)

  • Run:
    ros2 run my_py_pkg py_node

6. Errores comunes

  • Si hay un error de sintaxis en el archivo Python, el build fallará y se mostrará el mensaje en la consola.

  • Otros errores como excepciones de tiempo de ejecución solo se detectarán al ejecutar el nodo.

7. Resumen de nombres importantes

  • Nombre del archivo: my_first_node.py

  • Nombre del nodo: 'py_test' (definido en __init__)

  • Nombre del ejecutable: py_node (definido en setup.py)
    Estos nombres pueden coincidir, pero son conceptos distintos.
