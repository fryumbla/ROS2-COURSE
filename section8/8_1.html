8.1. Creación de un archivo Launch en ROS 2 (formato XML)

Aprenderás a crear tu primer archivo Launch en ROS 2 usando el formato XML, automatizando la ejecución de múltiples nodos.

📘 Notas de la clase

📦 Organización del proyecto

  • Es buena práctica crear un paquete separado para los archivos de lanzamiento, comúnmente llamado nombre_robot_bring_up.

  • Esto ayuda a mantener una estructura limpia y modular, especialmente al trabajar con múltiples paquetes y configuraciones.

📁 Creación del paquete bring_up

  1. Desde el directorio src/ de tu workspace:

    • Ejecuta ros2 pkg create nombre_robot_bring_up.

    • Ejemplo: ros2 pkg create my_robot_bring_up.

  2. Archivos/folders generados:

    • CMakeLists.txt y package.xml → mantener.

    • include/ y src/ → eliminar si no se van a compilar nodos en este paquete.

  3. Crear carpeta launch/ dentro del paquete:

    • Aquí irán todos los archivos .launch.xml.

⚙️ Configuración en CMakeLists.txt

  • El archivo CMakeLists.txt debe incluir la instrucción para instalar los archivos de lanzamiento:

install( DIRECTORY launch DESTINATION share/${PROJECT_NAME}/ )

  • No es necesario modificar package.xml para el build, pero sí para declarar dependencias de ejecución.

🔧 Añadir dependencias en package.xml

  • Si los launch files utilizan nodos de otros paquetes, declara las dependencias con:

<exec_depend>nombre_del_paquete</exec_depend>

Ejemplo:

<exec_depend>my_py_pkg</exec_depend>
<exec_depend>my_cpp_pkg</exec_depend>

🛠️ Creación del archivo XML de lanzamiento

  1. Crear un archivo en launch/, por ejemplo: number_app.launch.xml

  2. Estructura mínima del archivo:

<launch>
 <node pkg="my_py_pkg" exec="number_publisher" />
 <node pkg="my_cpp_pkg" exec="number_counter" />
</launch>

  1. Cada nodo se lanza usando una etiqueta <node> con atributos:

    • pkg: nombre del paquete donde está el ejecutable.

    • exec: nombre del ejecutable.

▶️ Ejecutar el archivo de lanzamiento

  1. Compilar el paquete:

colcon build --packages-select my_robot_bring_up

  1. Source del entorno:

source install/setup.bash

  1. Lanzar la aplicación:

ros2 launch my_robot_bring_up number_app.launch.xml

  1. Verificar nodos y tópicos activos:

ros2 node list
ros2 topic list
ros2 topic echo /number_count
