8.5. Uso de Namespaces en ROS 2 y archivos Launch

Aprenderás a organizar nodos usando Namespaces en tus archivos Launch para una jerarquía clara y evitar conflictos de nombres.

📘 Notas de la clase

🧠 ¿Qué es un Namespace?

  • Un namespace es un prefijo que se añade a los nombres de nodos, tópicos o servicios.

  • Ayuda a organizar y aislar componentes dentro de una aplicación ROS 2.

  • Muy útil para trabajar con múltiples robots o sistemas similares, evitando colisiones de nombres.

🖥️ Uso desde la línea de comandos

  1. Ejecutar un nodo con namespace:

ros2 run my_cpp_pkg number_publisher --ros-args -r __ns:=/test

  1. Verifica con:

ros2 node list → Verás /test/number_publisher
ros2 topic list → Verás /test/number (el tópico también se ve afectado)

⚠️ Importante sobre el uso de / (barra)

  • Si en el código defines un tópico con / al inicio (/number), el namespace no se aplicará.

  • Si defines el nombre sin / (number), ROS aplicará automáticamente el namespace.

📌 Recomendación: evita usar / al inicio de nombres en tu código si quieres que los namespaces funcionen correctamente.

🧪 Ejemplo práctico

Si defines el tópico como:

self.publisher_ = self.create_publisher(Int64, 'number', 10)

...y ejecutas con namespace /test, ROS convertirá el tópico a:

/test/number

Pero si lo defines como:

'/' + 'number' → entonces el resultado será simplemente /number, sin namespace.

🛠️ Uso de Namespace en archivo Launch (XML)

  1. Añadir el atributo namespace dentro del nodo:

<node pkg="my_py_pkg" exec="number_publisher" name="my_number_publisher" namespace="ABC">
 <remap from="number" to="my_number" />
 <param from="$(find-pkg-share my_robot_bring_up)/config/number_app.yaml" />
</node>

  1. Aplicar también a otros nodos para que compartan el mismo namespace:

<node pkg="my_cpp_pkg" exec="number_counter" name="my_number_counter" namespace="ABC">
 <remap from="number" to="my_number" />
</node>

📉 Evitar errores comunes con remap

  • Si usas /number (con slash) en el atributo from, no funcionará con namespaces.

  • Usa solo number:

<remap from="number" to="my_number" /> → correcto
<remap from="/number" to="/my_number" /> → puede fallar si hay namespaces

🧪 Verificación con herramientas ROS 2

  • ros2 node list → Verifica que los nodos están bajo el namespace /ABC

  • ros2 topic list → Verifica que el tópico se ha remapeado a /ABC/my_number

  • rqt_graph → Puedes agrupar por namespaces para una visualización más clara

🐍 Equivalente en archivo Python (launch.py)

  1. Añade el argumento namespace directamente:

Node(
 package='my_py_pkg',
 executable='number_publisher',
 name='my_number_publisher',
 namespace='ABC',
 remappings=[('number', 'my_number')],
 parameters=[config_path]
)

📌 Recuerda también no usar / al inicio del nombre del tópico si quieres que el namespace se aplique.

💡 Ventajas de usar namespaces

  • Aisla lógicamente los nodos de cada robot o módulo.

  • Facilita el uso de múltiples instancias del mismo nodo.

  • Mejora la escalabilidad y el mantenimiento del sistema.
