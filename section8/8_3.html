8.3. Renombrar nodos y remapear tópicos

Aprenderás a renombrar nodos dinámicamente y remapear tópicos dentro de archivos Launch para una configuración flexible.

📘 Notas de la clase

🧾 Renombrar nodos desde XML

  1. Dentro del tag <node>, agrega el atributo name:

<node pkg="my_py_pkg" exec="number_publisher" name="my_number_publisher" />
<node pkg="my_cpp_pkg" exec="number_counter" name="my_number_counter" />

  • Esto renombra los nodos en tiempo de ejecución.

  • Puedes verificarlo con ros2 node list o herramientas como rqt_graph.

🔄 Remapear tópicos desde XML

  1. Abre el tag <node> para poder incluir contenido anidado:

<node pkg="my_py_pkg" exec="number_publisher" name="my_number_publisher">
 <remap from="number" to="my_number" />
</node>

  1. Haz lo mismo para el segundo nodo:

<node pkg="my_cpp_pkg" exec="number_counter" name="my_number_counter">
 <remap from="number" to="my_number" />
</node>

✅ Así, ambos nodos utilizarán el nuevo tópico /my_number en lugar de /number.

▶️ Verificación

Después de compilar y lanzar:

  • ros2 topic list → Verás /my_number

  • rqt_graph → Confirmarás que publisher y subscriber están conectados al nuevo tópico

🐍 Equivalente en archivo Python (launch.py)

En un archivo launch.py, puedes hacer lo mismo usando argumentos:

Node(
 package='my_py_pkg',
 executable='number_publisher',
 name='my_number_publisher',
 remappings=[('number', 'my_number')]
)

Node(
 package='my_cpp_pkg',
 executable='number_counter',
 name='my_number_counter',
 remappings=[('number', 'my_number')]
)

  • El atributo name renombra el nodo.

  • El atributo remappings es una lista de tuplas con formato (from, to).

🔍 Aunque Python permite más lógica, el archivo XML es más compacto y directo:
8 líneas en XML vs 24 líneas en Python para esta misma funcionalidad.
