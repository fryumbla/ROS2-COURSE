8.4. Carga de parámetros desde archivos Launch en ROS 2 (directo y desde YAML)

Aprenderás a cargar parámetros en tus nodos directamente desde archivos Launch o desde archivos YAML externos.

📘 Notas de la clase

⚙️ Asignar parámetros directamente en XML

  1. Asegúrate de que el <node> esté abierto con etiquetas de inicio y cierre.

  2. Usa etiquetas <param> dentro del nodo, con los atributos name y value:

<node pkg="my_py_pkg" exec="number_publisher" name="my_number_publisher">
 <remap from="number" to="my_number" />
 <param name="number" value="6" />
 <param name="timer_period" value="1.5" />
</node>

✅ Los valores deben coincidir con los nombres definidos en el código del nodo.

📂 Organización del archivo YAML

  1. Crea un directorio config/ dentro del paquete my_robot_bring_up.

  2. Declara su instalación en CMakeLists.txt:

install(
 DIRECTORY launch config
 DESTINATION share/${PROJECT_NAME}/
)

  1. Crea un archivo YAML, por ejemplo number_app.yaml, con este contenido:

my_number_publisher:
 ros__parameters:
  number: 6
  timer_period: 1.5

📥 Cargar YAML desde el archivo Launch (XML)

En lugar de definir cada parámetro con <param>, puedes cargar el archivo completo:

<param from="$(find-pkg-share my_robot_bring_up)/config/number_app.yaml" />

✅ Esta línea debe ir dentro del nodo correspondiente (<node>).

✅ Verificación

  • ros2 topic echo /my_number → Muestra el valor publicado (6) con el intervalo especificado.

  • ros2 param get /my_number_publisher number → Confirma el valor (6)

  • ros2 param get /my_number_publisher timer_period → Confirma el valor (1.5)

🐍 Equivalente en archivo Python (launch.py)

  1. Parámetros manuales como diccionario (dentro del Node):

Node(
 package='my_py_pkg',
 executable='number_publisher',
 name='my_number_publisher',
 parameters=[{ 'number': 6, 'timer_period': 1.5 }]
)

  1. Para cargar desde un YAML:

  • Importa lo necesario:

from ament_index_python.packages import get_package_share_directory
import os

  • Define la ruta al archivo YAML:

config_path = os.path.join(
 get_package_share_directory('my_robot_bring_up'),
 'config',
 'number_app.yaml'
)

  • Y pásalo en el parámetro:

Node(
 package='my_py_pkg',
 executable='number_publisher',
 name='my_number_publisher',
 parameters=[config_path]
)

💾 Código trabajado / recursos

  • number_app.launch.xml: parámetros definidos directamente y mediante archivo YAML.

  • config/number_app.yaml: archivo de configuración de parámetros.

  • number_app.launch.py: versiones equivalentes usando diccionarios y carga desde YAML.
