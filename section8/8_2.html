8.2. Comparación y uso de archivos Launch en ROS 2 (XML vs Python)

Conocerás las diferencias entre archivos Launch en formato XML y Python, y aprenderás cuándo usar cada uno.

📘 Notas de la clase

🗂️ Lenguajes disponibles para archivos launch

ROS 2 permite escribir archivos launch en tres lenguajes:

  • XML ✅ (más utilizado y recomendado)

  • YAML ❌ (poco usado y propenso a errores, se omite)

  • Python ✅ (más flexible, pero más complejo)

🧠 Recomendación general

  • Usar XML siempre que sea posible: más simple, claro y suficiente para la mayoría de los casos.

  • Python se puede usar si se requiere lógica avanzada o se desea reutilizar código.

🐍 Creación de un archivo launch en Python

  1. Asegúrate de tener un paquete de lanzamiento creado (my_robot_bring_up) y con el folder launch/ configurado.

  2. Crea un archivo number_app.launch.py dentro de la carpeta launch/.

  3. Estructura básica del archivo Python:

from launch import LaunchDescription
from launch_ros.actions import Node

def generate_launch_description():
 number_publisher = Node(
  package='my_py_pkg',
  executable='number_publisher'
 )
 number_counter = Node(
  package='my_cpp_pkg',
  executable='number_counter'
 )
 ld = LaunchDescription()
 ld.add_action(number_publisher)
 ld.add_action(number_counter)
 return ld

✅ Este archivo inicia los mismos nodos que el XML, pero con mayor cantidad de líneas y sintaxis más compleja.

🔁 Incluir archivos Python desde XML

Si necesitas aprovechar una funcionalidad escrita en un archivo Python (launch.py) pero quieres mantener el control desde un archivo XML:

  1. Crea un archivo XML, por ejemplo: number_app_from_python.launch.xml

  2. Usa la siguiente estructura para incluir el archivo Python:

<launch>
 <include file="$(find-pkg-share my_robot_bring_up)/launch/number_app.launch.py" />
</launch>

✔️ Esto permite mantener la simplicidad del XML, pero incluir funcionalidades específicas escritas en Python.

▶️ Comandos útiles

  • Compilar el paquete:
    colcon build --packages-select my_robot_bring_up

  • Activar entorno:
    source install/setup.bash

  • Lanzar archivo Python:
    ros2 launch my_robot_bring_up number_app.launch.py

  • Lanzar archivo XML que incluye uno Python:
    ros2 launch my_robot_bring_up number_app_from_python.launch.xml

💾 Código trabajado / recursos

  • number_app.launch.py: archivo launch en Python que inicia dos nodos.

  • number_app_from_python.launch.xml: archivo XML que incluye un launch Python.
